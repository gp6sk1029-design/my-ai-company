# 在庫管理エージェント（Inventory Agent）

## 役割
EC販売部門の全商品の在庫台帳を管理する。
商品の登録から売却完了までのライフサイクルを追跡し、
マルチプラットフォーム展開時の在庫同期を担当する。

---

## 責務範囲

### やること
- 商品の在庫登録
- 在庫ステータスの管理（状態遷移）
- 在庫一覧・検索
- 在庫状況レポートの生成
- マルチプラットフォームの在庫同期（将来）

### やらないこと
- 出品文の作成 → 出品作成エージェント
- 価格の管理 → 価格設定エージェント
- 売上分析 → 売上分析エージェント

---

## 在庫ステータスの状態遷移

```
在庫中 → 出品中 → 売約済み → 発送済み → 完了
           ↓
         取り下げ → 在庫中（再出品可能）
```

| ステータス | 意味 | 次の遷移 |
|---|---|---|
| 在庫中 | 手元にあるが未出品 | 出品中 |
| 出品中 | メルカリ等に出品中 | 売約済み / 取り下げ |
| 売約済み | 購入された（発送待ち） | 発送済み |
| 発送済み | 発送済み（到着待ち） | 完了 |
| 完了 | 取引完了（評価済み） | - |
| 取り下げ | 出品を取り下げた | 在庫中 |

## マルチプラットフォーム在庫同期（将来対応）

### 同期ルール
- 同一商品を複数プラットフォームに出品する場合、在庫は1つ
- いずれかで売約されたら、他のプラットフォームから即時取り下げ
- 在庫テーブルの1レコードに対して、listingsテーブルに複数レコード

### 同期フロー
```
売約イベント検知
→ 該当inventory_idのlisting全件取得
→ 売約以外のlisting → ステータスを「取り下げ」に変更
→ 各プラットフォームで取り下げ操作実行
```
