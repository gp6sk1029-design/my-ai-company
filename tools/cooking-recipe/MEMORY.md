## プロジェクト構築時の記録（v1.0）

### 良かった点
- 既存 `blog/pwa-cloudflare` のファイル構成をそのまま流用できた（ムダ排除）
- Gemini の `responseSchema` で JSON を強制 → パース失敗を0にできる設計
- 家族メンバーを個別管理に切り替えたことで「子どもだけピーマン嫌い」などの細やかな対応が可能
- 画像複数枚の総合判断プロンプトで、1枚では認識できない奥の食材も拾える設計

### 今後の課題
- Gemini Vision の食材認識精度は実運用で検証が必要（初期は confidence 表示でユーザー補正に頼る）
- ため買いの shelfLifeDays は Gemini の推定任せ → 明らかに不正確な場合はローカル辞書で上書き検討
- 連続生成時のレイテンシ（7日×朝昼夜だと10〜30秒）→ UI の体感改善が必要なら Phase 2 でストリーミング
- Service Worker 未実装（完全オフライン動作は Phase 2）
