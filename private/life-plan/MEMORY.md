### 経緯

  - 2026-04-25に発覚し、抜本再設計（下記）

## 2026-04-25 MF連携を抜本再設計（by main agent）

### 経緯
- ユーザー申告「マネーフォワードのCSV情報は家計簿の情報しか登録できないので現状ツールではつかえない」
- 既存の「MF資産CSV取込」機能は前提崩壊（動かない状態で本番公開されていた）

### 採用した構成（ハイブリッド3方式）
1. **スクショOCR取込**（資産タブ）：Tesseract.js（`jpn+eng`）を動的ロード、端末内OCR。`¥([\d,]+)` で金額抽出し、キーワードで `kind` 分類
2. **一括棚卸しモード**（資産タブ）：OCR失敗時の現実解。全口座を1モーダルで順入力、差分(±円/±%)を即時表示
3. **家計簿CSV取込**（収支タブ）：`計算対象=1 ∧ 振替=0 ∧ 金額<0` のみ集計、大項目×月で直近3ヶ月平均→ `expense[]` にマッピング更新

### 設計判断のポイント
- Tesseract.js を採用（Cloudflare Workers AI / Vision API 不採用）
  - 理由：IndexedDB思想（端末内完結・外部送信なし）との整合性、API費用ゼロ
  - 精度不足は差分プレビュー＋手動修正で補償
- `mfSnapshots` ストアは削除せず OCR履歴保管用に転用（棚卸しの証跡）
- 家計簿CSVは Shift_JIS / UTF-8 両対応（MF実態）
- `window.LIFEPLAN_MF` に純粋関数を露出（parseKakeiboCsv / summarizeKakeibo / extractBalancesFromOcr / classifyAssetLine）→ Node単体テスト可能に

### 成功パターン
- 事実と違う前提で動く「動かないコード」は、放置せず完全撤去→用途転換で情報を失わない設計にする
- ユーザー提案（スクショ取込）を素直に採用し、OCR失敗時の下位フォールバック（手動棚卸し）を必ず用意
- 家計簿CSVを「支出予測の精度上げ」に活用することで、動かない旧機能を単に削除するのではなく価値を上積み
