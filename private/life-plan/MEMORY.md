## 2026-04-23 初期構築（by main agent）

### 経緯
- ユーザーの既存Google Driveライフプランシート（xlsx）を刷新してPWA化
- 足りない要素：マネーフォワード連携／子供年齢別教育費／ライフイベント別CF／投資複利

### 採用した構成
- `private/cooking-recipe`（献立くん）と同じCloudflare Pages + Functions + IndexedDBスタック
- 計算ロジックを `calc.js` に純粋関数で分離（テスト容易化）
- 6タブ構成（ホーム・基本・収支・教育・資産・イベント）

### 成功パターン
- 教育費の出典（文科省＋日本政策金融公庫）を `_source` フィールドに明記→将来更新時に迷わない
- MFはCSVインポート方式に割切り（公開API不在の現実解）
- 新NISAの生涯1,800万円capを超えた分を特定口座に自動オーバーフローさせる実装→税務的に実態に近い

### 注意点・改善余地
- `buildCashflow` 内の子供年齢→親年齢写像が冗長（整理予定）
- 入力UIがまだ `prompt()` ベース→スマホ操作性を上げるためフォーム化したい
- Chart.jsのstack設定が datasets 付与後の後処理になっている→初期化時に指定する形に整理
- ユーザー提供の元xlsxが到着したら `reference/` に配置し、項目ギャップを埋める

### ROI評価
- 投じた時間：本セッション内で実装完了
- 得られたリターン：年1〜2回しか見直していなかったライフプランを「スマホで数分でシミュ」にできる→家計判断の高速化
- 次回改善：wrangler pages dev での動作確認→見つかったバグを即 calc.js に単体テスト化
